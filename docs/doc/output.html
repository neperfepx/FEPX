Simulation Results (simulation.cfg)

Simulation results include nodal results, elemental results, mesh face results, simulation restart information, and other, advanced output. The printing of the field variable files is defined as follows:

print <result1> <result2> ...

where <result1>, <result2>, etc. are the particular result fields to be output. Generally, and particularly for nodes and elements, outputs are provided per line (one node or element, etc. per line) and are space delimited within a line. Note that, elemental quantities are internally calculated at each Gauss quadrature point within the element (15 total), but are only printed as one value per element, which is that associated with the quadrature point that falls at the element centroid.

The results are written in a Simulation Directory (.sim), where each result is written in its own file, at each step (one file for each result and each step). Results can have scalar values, vectorial values or tensorial values. In the result files, values for the different entities (nodes or elements) are written on successive lines, with components written on successive columns (space delimited). The components are written as follows:

  • scalar, \(a\): \(a\);

  • vector, \(v\): \(v_1\), \(v_2\), \(v_3\);

  • symmetrical tensor, \(t\): \(t_{11}\), \(t_{22}\), \(t_{33}\), \(t_{23}\), \(t_{31}\), \(t_{12}\) (Voigt notation);

  • skew-symmetrical tensor, \(t\): \(t_{12}\), \(t_{13}\), \(t_{23}\);

  • non-symmetrical tensor, \(t\): \(t_{11}\), \(t_{12}\), \(t_{13}\), \(t_{21}\), \(t_{22}\), \(t_{23}\), \(t_{31}\), \(t_{32}\), \(t_{33}\).

Nodal Results

Coordinates (coo)

Position vector, written in the sample basis.

Velocities (vel)

Velocity vector, written in the sample basis.

Displacements (disp)

Displacement vector, written in the sample basis.

Elemental Results

Crystallographic Orientation (ori)

Orientation written using the same parameterization as that used in input, which can be

  • the Rodrigues vector, \({\bf r} = {\bf t} \tan{(\omega / 2)}\);

  • the Euler-Bunge angles, \((\varphi_1,\,\phi,\,\varphi_2)\) expressed in degrees;

  • the Euler-Kocks angles, \((\Psi,\,\Theta,\,\phi)\) expressed in degrees;

  • the axis-angle pair, \((t_1,\,t_2,\,t_3,\,\omega)\), where \(\bf{t}\) is the (unit) axis of rotation and \(\omega\) is the angle of rotation about said axis, expressed in degrees;

  • the quaternion, \((q_0,\,q_1,\,q_2,\,q_3)\).

Slip System Resolved Shear Stress (rss)

\(\tau^\alpha\) scalar values, printed in the order specified in Slip Systems.

Slip System Critical Resolved Shear Stress(es) (crss)

\(g_0\) scalar value(s), printed in the order specified in Slip Systems. For isotropic hardening, only one value is printed.

Slip System Shear Rate (sliprate)

\(\dot\gamma^\alpha\) scalar values, printed in the order specified in Slip Systems.

Slip System Slips (slip)

\(\gamma^\alpha\) scalar values, printed in the order specified in Slip Systems.

Stress Tensor (stress)

\(\bf \sigma\) symmetrical tensor, written in the sample basis.

Equivalent Stress (stress_eq)

\(\sigma_{eq}\) scalar value, calculated based on the deviatoric portion of the stress tensor, \({\bf \sigma}^{\prime}\), via the tensor inner product: \(\sigma_{eq} = \sqrt{ {3 \over 2} {\bf \sigma}^{\prime} : {\bf \sigma}^{\prime}}\)

Total Strain Tensor (strain)

\(\bf E\) symmetrical tensor, written in the sample basis.

Equivalent Total Strain (strain_eq)

\(E_{eq}\) scalar value, calculated based on the deviatoric portion of the total strain tensor, \({\bf E}^{\prime}\), via the tensor inner product: \(E_{eq} = \sqrt{ {2 \over 3} {\bf E}^{\prime} : {\bf E}^{\prime}}\)

Elastic Strain Tensor (strain_el)

\(\bf E^e\) symmetrical tensor, written in the sample basis.

Equivalent Elastic Strain (strain_el_eq)

\(E^e_{eq}\) scalar value, calculated based on the deviatoric portion of the elastic strain tensor, \({\bf E}^{e \prime}\), via the tensor inner product: \(E^e_{eq} = \sqrt{ {2 \over 3} {\bf E}^{e \prime} : {\bf E}^{e \prime}}\)

Plastic Strain Tensor (strain_pl)

\(\bf E^p\) symmetrical tensor, written in the sample basis.

Equivalent Plastic Strain (strain_pl_eq)

\(E^p_{eq}\) scalar value, calculated based on the deviatoric portion of the elastic strain tensor, \({\bf E}^{p \prime}\), via the tensor inner product: \(E^p_{eq} = \sqrt{ {2 \over 3} {\bf E}^{p \prime} : {\bf E}^{p \prime}}\)

The below variables are of secondary general interest:

Velocity Gradient Tensor (velgrad)

\({\bf L}^p\) general tensor, written in the sample basis.

Deformation Rate Tensor (defrate)

\(\bf D\) symmetrical tensor, written in the sample basis.

Equivalent Deformation Rate (defrate_eq)

\(D_{eq}\) scalar value, corresponding to the \(\bf D\) tensor inner product: \(D = \sqrt{ {2 \over 3} {\bf D} : {\bf D} }\)

Plastic Deformation Rate Tensor (defrate_pl)

\({\bf D}^p\) symmetrical tensor, written in the sample basis.

Equivalent Plastic Deformation Rate (defrate_pl_eq)

\(D^p_{eq}\) scalar value, corresponding to the \(\bf D^p\) tensor inner product: \(D = \sqrt{ {2 \over 3} {\bf D^p} : {\bf D^p} }\)

Plastic Spin Rate Tensor (spinrate)

\({\bf W}^p\) skew-symmetric tensor, written in the sample basis.

Work (work)

\(W\) Scalar value, calculated as the time integration of the tensor inner product of the deformation rate tensor and the Cauchy stress tensor: \(W = \int{ (\sigma : {\bf D}) }\Delta t\).

Plastic Work (work_pl)

\(W^p\) Scalar value, calculated as the time integration of the tensor inner product of the plastic deformation rate tensor and the deviatoric portion of the Cauchy stress tensor: \(W^p = \int{ ( \sigma ^ \prime : {\bf D }^ p ) \Delta t}\).

Work Rate (workrate)

\(\dot{W}\) scalar value, calculated as the tensor inner product of the deformation rate tensor and the Cauchy stress tensor: \(\dot{W} = \sigma : {\bf D}\).

Plastic Work Rate (workrate_pl)

\(\dot{W}^p\) scalar value, calculated as the tensor inner product of the plastic deformation rate tensor and the deviatoric portion of the Cauchy stress tensor: \(\dot{W}^p = \sigma ^ \prime : {\bf D }^ p\)

Lattice Reorientation Rate Vector (rotrate)

(reorientation axis x reorientation velocity) vector, written in the sample basis.

Spin Part of the Lattice Reorientation Rate Vector (rotrate_spin)

(reorientation axis x reorientation velocity) vector, spin part, written in the sample basis.

Slip Part of the Lattice Reorientation Rate Vector (rotrate_slip)

(reorientation axis x reorientation velocity) vector, slip part, written in the sample basis.

Surface Results

Integrated Forces (force)

Integrated forces (or “loads”) for all surfaces, provided at each increment and written as post.force.* files, which are constructed via the defined faset_label strings in the simulation.msh file (for a cubic domain, x0, x1, y0, y1, z0 and z1, where 0 and 1 represent the minimum and maximum coordinates). These files are generally tabulated with the given structure:

<step> <incr> <force_x> <force_y> <force_z> <surf_area> <current_time>

where <step> is the prescribed load step, <incr> is the total increment value being printed, <force_x> is the surface-integrated force in the x direction, <force_y> is the surface-integrated force in the y direction, <force_z> is the surface-integrated force in the z direction, <surf_area> is the current surface area of the given face, and <current_time> is the total simulated time at the time of printing.

If multiaxial loading is utilized, an additional <length> column will be appended to the right of <current_time. The <length> column contains the maximal coordinate values of the domain and these values are stored in their associated face files. For example, the maximal mesh coordinate value in the x direction is stored in the post.force.x0 and post.force.x1 files accordingly.

Advanced Results

Convergence Information (convergence)

Convergence statistics, provided at each increment and written as a post.conv file, which is tabulated with the given structure:

<incr> <iter> <nr> <r_norm> <rx_norm> <f_norm> <delu_norm> <delux_norm> <u_norm> <cg_iter>

where <incr> is the total increment value being printed, <iter> is a sub-increment iteration, <nr> is a boolean that notifies if the given iteration was a Newton-Raphson iteration, <r_norm> is a residual norm, <rx_norm> is the maximum absolute value of <r_norm>, <f_norm> is a force norm, <delu_norm> is the change in velocity norm, <delux_norm> is the maximum absolute value of <delu_norm>, <u_norm> is the velocity norm, and <cg_iter> is the number of iterations the conjugate gradient solver performed. All norms are computed as 2-norm or the square root of the inner product of a vector by itself.

Restart Information (restart)

Restart files, which allows to restart a simulation at the last completed step. Restart files are only written at the end of a successful step and not at individual increments.
